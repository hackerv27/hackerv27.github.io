RSA回顾

RSA公钥密码体制描述如下：
1）选取两个大素数p和q，p和q保密
2）计算n=pq，φ(n)=(p-1)(q-1)，n公开，φ(n)（欧拉函数值）保密
3）随机选取正整数 1<e<φ(n)，满足gcd(e,φ(n)) = 1，e是公开的加密密钥
4）计算d，满足ed≡1(modφ(n))，d是保密的解密密钥

加密变换：c = m^e mod n
解密变换：m = c^d mod n

满足gcd(e,φ(n)) = 1

os：平常解题都是默认成立的，拿起来就解，没有多想🙏

证明：gcd(a,n)=1是∃b,ab≡1(mod n)的充要条件

充分性：

由gcd(a,n)=1,由裴蜀定理：存在整数x,y，使得ax+ny=gcd(a,b)=1

上式可化为ax≡1(modn)，得证

必要性：

假设x是a对于b的乘法逆元，则有 a∗x=1modb, 即x是方程 a∗x+b∗y=1的一个整数解。 但是形如a∗x+b∗y的最小正整数为gcd(a,b)。 此结论可以通过扩展欧几里得算法（EEA）的正确性得到。 而1又是最小的非零自然数，所以只有当gcd(a,b)=1时，a才有对于b的乘法逆元。

因此，在RSA算法中，对任意e，存在d的充要条件是：gcd(e,ϕ(N))=1

三种情况

m^gcd(e,phi)^<n

这也很好理解，有点算是低加密指数攻击的变型

原来我们的加密公式是c≡me(modn)c≡me(modn)

现设t=gcd(e,phi)t=gcd(e,phi)

那么加密公式变为c≡m^e∗t(modn)c≡me^∗t(modn)，其中^e∗t=ee^∗t=e

显然此时gcd(^e,phi)=1gcd(e^,phi)=1

我们可以先利用^e,phie^,phi算出的私钥^dd^进行解密，求出mtmt

然后由于mt<nmt<n，我们直接对mtmt开tt次根就好了。

e=2这种特殊情况对应着的是rabin密码系统，具体原理可移步密码学学习笔记 之 数论四大定理及应用

这里要求的条件比较苛刻，你需要知道nn的分解p,qp,q然后再在模p,qp,q下对cc开ee次根，也即域下开根，现存有一些算法可以做到，比较有效的是AMM，我[这篇文章](https://jayxv.github.io/2019/12/04/密码学学习笔记之浅析On r-th Root Extraction Algorithm in Fq/)给的是另一种算法，稍微麻烦些，要分类讨论。最后如果m>p，m>qm>p，m>q的话还要CRT一波，解集大小是gcd(e,p−1)∗gcd(e,q−1)gcd(e,p−1)∗gcd(e,q−1)。对应题目可参考2019NCTF-EASYRSA
