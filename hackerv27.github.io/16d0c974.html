签到电台/C

“弼时安全到达了”所对应的7个电码：
1732 2514 1344 0356 0451 6671 0055

模十算法示例：1732与6378得到7000

发包示例：/send?msg=s

与密码本模10运算，得到2979481690868655519524457577

然后发包

key：当时模10运算后，不知道怎么去敲电码/(ㄒoㄒ)/~~，还傻乎乎的用鼠标硬敲……

tip：要用burpsuite抓包（web tool)

通过burpsuite抓包发送两次得到flag（交给web手吧🤗）

基于挑战码的双向认证1、2

非预期解

1、题目描述是ssh端口，直接ssh登录上去，发现cube-challenge文件

2、进去后没发现什么有用的东西

3、利用正则寻找flag：find /| grep flag

4、在/root/cube-shell/instance/flag_server/发现flag1.txt和flag2.txt

5、尝试访问，发现flag没有设置访问权限，两个flag到手

直接grep -r “flag{” /

就能找到flag

FinalShell（国产ssh连接工具）

基于挑战码的双向认证3

1、同样ssh连接，故技重施一波

2、找到flag位置，尝试访问，提示没有权限访问

3、查看当前linux版本，发现是Red Hat 4.8.5

4、尝试利用弱口令进行root提权，发现密码为toor，直接拿到flag

Iso9798

连接后通过sha256密码碰撞得到前四位输入进行交互

随机输入数字得到回显的加密密文

将密文分成三组，交换前两组位置后进行提交得到flag
